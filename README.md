# Security Foundations: Governance & Frameworks

## CIA Triad

The CIA Triad represents the three core principles of information security:

- **Confidentiality** - Ensuring that sensitive information is accessible only to authorized individuals.
- **Integrity** - Protecting data from unauthorized modification or tampering.
- **Availability** - Ensuring systems and data are accessible when needed by authorized users.

These principles are the foundation for designing security policies, technologies, and risk management strategies.

## AAA Framework

The AAA Framework stands for:

- **Authentication** - Verifying the identity of a user or system.
- **Authorization** - Determining what an authenticated user is allowed to access or perform.
- **Accounting** - Tracking and logging user actions for auditing and monitoring purposes.

AAA ensures controls access to systems and provides accountability within secure environments.

## Governance and Cybersecurity Roles

Governance shapes cybersecurity roles by defining policies, standards, risk tolerance, and compliance requirements within an organization. It determines how security strategies align with business objectives and regulatory obligations. Cybersecurity professionals must operate within governance structures to ensure that technical controls support legal, ethical, and operational standards. Without governance, security efforts become reactive and inconsistent. Effective governance ensures accountability, structured decision-making, and sustainable security programs.

# Lab Infrastructure & Virtualization Setup

## What is a Hypervisor?

A hypervisor is software or firmware that enables multiple operating systems to run on a single physical machine by managing and allocating hardware resources to virtual machines. It abstracts the physical hardware and allows isolated environments to operate simultaneously.

## What is a Virtual Machine (VM)?

A virtual machine is a software-based emulation of a physical computer. It runs its own operating system and applications while sharing the host system's hardware resources through a hypervisor.

## Why Isolation Matters in Cybersecurity

Isolation ensures that activities inside one virtual environment do not impact the host system or other virtual machines. This containment is critical when testing malware, analyzing vulnerabilities, or experimenting with configurations that may be unstable or insecure. Isolation minimizes risk to production systems and prevents unintended spread of malicious code.

## Virtualization and the CIA Triad

Virtualization supports **Confidentiality** by isolating sensitive workloads from other systems. It preserves **Integrity** by allowing secure snapshots and rollback features that protect against unwanted system modifications. It enhances **Availability** through resource allocation, failover capabilities, and disaster recovery solutions. By segmenting environments and controlling access, virtualization strengthens enterprise security architecture.
